# Oracle Review: WPHX-315 Bridge Router Package Roots

Date: 2026-07-03

Reviewer: external architecture oracle (GPT 5.5 Pro)

Review target: the WPHX-315.06 copied-package-root plus deterministic bridge-router pattern, and whether WPHX-315 can close as a bounded admin common/list-table evidence slice.

## Verdict

The copied-package-root plus deterministic-router pattern is sound as temporary bridge evidence, but only with a narrowed claim. WPHX-315 may close as a bounded evidence slice after WPHX-315.06 records package topology and bridge-router observation parity instead of broad installed WordPress behavior parity.

## Key Finding

WPHX-315.06 mirrors 24 selected upstream WordPress admin/common/list-table PHP files into regenerated oracle and candidate package roots, but the router emits deterministic JSON/HTML observations itself. For the seven HTTP cases, it does not dispatch into mirrored `wp-admin/index.php`, `wp-admin/edit.php`, `wp-admin/admin.php`, `wp-admin/tools.php`, or `wp-admin/update-core.php`.

Those mirrored files are package-topology, provenance, and future overlay targets. They are not executed route implementations for the WPHX-315.06 installed-style HTTP gate.

## Accepted Actions

- WPHX-315.06 now records `behavior_parity_claimed: false`.
- WPHX-315.06 now records separate router observation and package topology claims.
- WPHX-315.06 now records explicit non-claims for mirrored upstream source execution, installed WordPress route execution, generated candidate overlays, and durable original-path adapter ownership.
- The bridge router now carries a PHP comment marking it as test-harness-only code, not distributable runtime logic.
- The WPHX-315.06 receipt summary and non-claims now say the router is fixture observation code, not a WPHX-owned admin template or implementation artifact.
- WPHX-315 domain and parent closure receipts keep WPHX-315.06 as package-topology and bridge-router evidence only.
- `wordpresshx-u87c` / `WPHX-315.08` tracks the typed HXX/HHX admin markup pilot.
- `wordpresshx-fyqs` / `WPHX-315.09` tracks generated-overlay manifests and durable-claim guardrails for future copied-package-root bridge gates.

## Policy

Copied package roots plus deterministic routers are allowed as temporary bridge evidence only. They cannot support public PHP replacement, durable original-path ownership, whole-file ownership, installed route execution, or broad behavior parity claims by themselves.

Future candidate package divergence requires an explicit generated overlay manifest that records overlay paths, source, adapter/compiler evidence, replaced upstream hashes, and unexpected candidate package differences.

Future admin or template-heavy closures should not lean mainly on copied package roots plus deterministic router observations. The next adjacent closure needs stronger evidence, such as generated WPHX PHP original-path adapters, compiler-emitted segment shells, a typed HXX/HHX markup fixture, real installed route execution through mirrored upstream files, browser/e2e evidence, or database-backed installed admin evidence appropriate to the claimed boundary.
